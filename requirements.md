# コールドウォレットにおける署名とスマートコントラクトを用いたアドレス改ざん防止策の提案

## 概要
本研究は、暗号資産の送金時において、コールドウォレットの送金先アドレスが改ざんされるリスクを低減するための新しい手法を提案するものである。ドメイン名と送金先アドレスを紐づけたデジタル署名と、スマートコントラクトによるアドレス解決を組み合わせることで、送金元が安全かつ確実に正しい送金先アドレスを検証できる仕組みを構築する。

## 研究内容
コールドウォレットは秘密鍵をオフラインで管理するため高いセキュリティを誇るが、実際の送金指示を生成する際、マルウェアやクリップボードハイジャックなどによって送金元デバイス上で送金先アドレス（`addr`）が書き換えられる脅威が存在する。本研究では、このアドレス改ざん攻撃を防ぐため、人間が識別可能なドメイン名（`domain`）を利用し、送金先アドレスの正当性をシステム的に検証可能なアーキテクチャの設計を行う。

## 提案手法
本手法は、サーバによる署名生成とスマートコントラクト上のデータ登録・照会機能によって構成される。具体的なプロセスは以下の通りである。

1. **署名の取得**
   送金先となるクライアントは、自身の `domain` と `addr` をサーバに送信し、サーバからその組み合わせに対するデジタル署名 $sig(domain, addr)$ を取得する。
2. **スマートコントラクトへの登録**
   送金先クライアントは、取得した $sig$ と `addr` を、`domain` に紐づく値としてスマートコントラクトに登録する。
3. **アドレスの解決と検証**
   オンラインデバイスはスマートコントラクトから `domain`, `addr`, `sig` を取得し、トランザクションを作成する。
4. **コールドウォレットへ転送**
   オンラインデバイスは `tx(addr, amount)`, `domain`, `sig` をコールドウォレットへ転送する。
5. **トランザクションへの署名**
   コールドウォレットはまず、`addr`, `domain` に対する署名 `sig` を検証し、`addr` と `domain` の対応関係を確認する。その後、`addr` を `domain` に変換し、コールドウォレットの画面上に表示を行う。ユーザは目視で `domain` を確認し、送金先が正しいことが確認出来たらコールドウォレットでのトランザクションへの署名を許可する。

## 基本設計（アーキテクチャ）
本システムは、アドレス改ざんを防止するために以下の5つの主要コンポーネントで構成される。

### 1. 送金先クライアント (Payee Client)
- 資金を受け取る側が操作する端末・アプリケーション。
- 管理するアドレス（`addr`）と自身を証明するドメイン名（`domain`）を署名サーバへ送信し、署名（`sig`）を要求する。
- 取得した `sig` と `addr` を、`domain` の検証情報としてスマートコントラクトに登録するトランザクションを発行する。

### 2. 署名サーバ (Signature Server)
- 信頼された第三者、またはサービス提供者が運営するサーバ。
- クライアントから送信された `domain` と `addr` の関係性を確認し、サーバ自身の秘密鍵を用いてデジタル署名 $sig(domain, addr)$ を生成し、クライアントに返却する。
- サーバの公開鍵は、後段のコールドウォレット等に安全な方法で配布されていることを前提とする。

### 3. スマートコントラクト (Smart Contract)
- ブロックチェーン上にデプロイされたレジストリ（アドレス解決機能）。
- `domain` をキーとして、対応する `addr` と `sig` を不変の公開データとして保存する。
- 送金元のオンラインデバイスからの要求に対し、正しい `addr` とその検証用 `sig` を提供する。

### 4. オンラインデバイス (Online Device / 送金元アプリ)
- インターネットに接続された、送金者が操作する端末（スマートフォンアプリなど）。
- 送金先として指定された `domain` または `addr` からアドレス解決を行い、スマートコントラクトから `addr`, `domain`, `sig` を取得する。
- 取得した `addr` を宛先、入力された金額を `amount` とした未署名トランザクション（`tx`）を作成し、`tx`, `domain`, `sig` の3点をQRコードでオフラインのコールドウォレットへ送信する。

### 5. コールドウォレット (Cold Wallet)
- インターネットから物理的に隔離された環境で送金元の秘密鍵を管理し、トランザクションへの署名を行うデバイス。
- オンラインデバイスから受け取った未署名トランザクション `tx` の宛先 `addr` と、付随する `domain`、および `sig` を読み込む。
- 内部に保持する署名サーバの公開鍵を用いて署名 `sig` を検証し、「提示された `addr` が確実にこの `domain` に紐づくものである」ことを暗号学的に担保する。
- 検証成功後、無機質なアドレス単体の代わりに、人間が識別できる `domain` を画面上に表示する。
- ユーザによる画面上での目視確認後、トランザクション `tx` に対してオフライン内で署名を行い、署名済みトランザクションを出力してオンラインデバイスへ戻す。
