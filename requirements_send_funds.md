# 要件定義書：ドメイン解決および送金機能

## 1. 目的
登録済みのドメイン名を用いてスマートコントラクトからアドレスを解決し、そのアドレスに対して送金を行う機能を実現する。

## 2. 機能要件

### 2.1. ドメイン名の入力と解決
- ユーザは送金先として「ドメイン名」を入力する。
- アプリはスマートコントラクト（SimpleENS等）の `resolve` 関数を呼び出し、入力されたドメイン名に対応する「アドレス」を取得する。
- 解決中にユーザが待機できるよう、ローディング状態を表示する。

### 2.2. 送金先の確認と金額入力
- 解決された「アドレス」を画面上に表示し、ユーザが正誤を確認できるようにする。
- ユーザは「送金金額（Ether単位）」を入力する。

### 2.3. 送金トランザクションの作成と実行
- 解決されたアドレスおよび入力された金額をもとに、Ether送金用のトランザクションを作成する。
- トランザクションをネットワーク（Sepoliaテストネット等）へ送信し、成功時はトランザクションハッシュを表示する。

## 3. 非機能要件・検討事項
- **エラーハンドリング**:
    - **ドメイン名が解決できない（未登録）場合の挙動**:
        - 解決結果が `0x0000...`（零アドレス）や空文字の場合，「このドメインは登録されていません」といったエラーメッセージを表示する。
        - 送金先の確定を防ぐため、送金ボタンを無効化する。
    - 送金金額が不正、または残高不足の場合のエラー表示。
    - ネットワークエラー時のリトライや通知。
- **セキュリティ**:
    - 送金前に解決後のアドレスを明確に表示し、ユーザの最終確認を促す。
- **UX**:
    - 単位（Ether/Wei）の変換を適切に行う。
